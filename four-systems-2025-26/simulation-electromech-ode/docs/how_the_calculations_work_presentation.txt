================================================================================
HOW THE CALCULATIONS WORK — FOR PRESENTING TO YOUR MATLAB TEACHER
(Step-by-step: what we actually did to get efficiency, net energy, and ANOVA)
================================================================================

This is a walkthrough of the calculation flow so you can explain it in class.
It's not a glossary—it's "first we did this, then we did that, and that's how we got the numbers."


---- PART 1: THE FOUR-SYSTEM COMPARISON (analysis_compare_systems.m) ----

This is the main script that compares the four gravity storage systems. Everything below happens in order.


STEP 1 — Set up time for each system
-------------------------------------
We need a list of time points for one full cycle (lift + maybe hold + drop) for each system. We don't solve differential equations here; we just decide how long each phase lasts and create a time vector.

• We use linspace(0, T_total, N). For example for the dual-weight system: T_total = T1_d + T2_d (descent time + ascent time), and we use 250 points. So t2 is 250 numbers from 0 to the end of the cycle.

• So: one time vector per system (t2, t3, t4, t5), each with hundreds of points. That's our "discrete time" for the rest of the math.


STEP 2 — Know which phase we're in at each time (logical indexing)
-------------------------------------------------------------------
For each time point we need to know: "Are we in the descent phase? The ascent phase? The pump phase?" We use comparisons that give true/false for every time point.

• Example: ix1 = (t2 <= T1_d) means "true for all times during the first phase (descent), false after." ix2 = (t2 > T1_d) is true for the ascent phase.

• So we get logical vectors (ix1, ix2, ix_des5, ix_asc5, etc.) that tell us "at this index we're in this phase." We use them later to fill in velocity and power only where they apply.


STEP 3 — Velocity at each time (smooth profiles, element-wise math)
---------------------------------------------------------------------
We don't have a differential equation for position; we define velocity directly as a function of "how far through the phase we are." That gives smooth acceleration and deceleration (no jerks).

• We use a normalized time "tau" in each phase. For descent: tau1 = t2(ix1) / T1_d, so tau goes from 0 to 1 over the descent. Then we set:
    v_cab(ix1) = -1.35 * 4*tau1.*(1-tau1)
  So only the indices where ix1 is true get a velocity; the rest stay zero (we preallocated with zeros(size(t2))).

• The formula 4*tau*(1-tau) is a parabola: zero at start and end, maximum in the middle. So the weight speeds up then slows down. The .* is element-wise multiplication so we do that for every time point in the phase at once.

• We do the same idea for each system: different formulas for lift vs drop, and we use min(max(tau_d,0),1) sometimes to clamp tau between 0 and 1 so we don't get invalid values at the boundaries.


STEP 4 — Mechanical power from velocity (physics)
--------------------------------------------------
Power from gravity is force × velocity. Force is mass × g. So mechanical power (at the drum or at the weight) is something like P_mech = m * g * (-v) with a sign convention so that when the weight is going down we get positive power (into the system).

• Example dual-weight: P_mech = m_net * g * (-v_cab). So for every time point we have a value of P_mech. That's still a vector of 250 numbers.

• For buoyancy we have separate terms: motor power and pump power during lift, and generator power during drop. We fill those using the same idea: P_motor3(ix_l) = 50 + 80*tau_l.*(1-tau_l), and so on. So by the end of this step we have vectors of power (consumption and generation) over time for each system.


STEP 5 — Electrical power from mechanical (efficiency)
------------------------------------------------------
We convert mechanical power to electrical: when we're generating we multiply by an efficiency (e.g. eta_gen), when we're consuming we divide by an efficiency (e.g. eta_drive). So we get electrical power at each time.

• Example dual-weight:
  - During descent (ix1): we generate, so P_elec2(ix1) = eta_gen * max(P_mech(ix1), 0). The max(..., 0) keeps only positive mechanical power (generation).
  - During ascent (ix2): we consume, so P_elec2(ix2) = -max(0, -P_mech(ix2)) / eta_drive. So we get negative electrical power (consumption), and we might store it as positive "consumed" when we integrate.

• To get total generated energy we'll integrate only the positive part of P_elec; for consumed we integrate the negative part (or the absolute value). So we use things like max(P_elec2, 0) for generation and max(-P_elec2, 0) for consumption.


STEP 6 — Energy from power: integrate over time (trapz)
------------------------------------------------------
Energy is the integral of power over time. We have power at many time points, so we approximate the integral with the trapezoidal rule.

• trapz(t, P) does that: it takes the time vector t and the power vector P (same length) and returns one number—the area under the curve P vs t. So:
  E_gen_2 = trapz(t2, max(P_elec2, 0))   % total electrical energy generated
  E_cons_2 = trapz(t2, max(-P_elec2, 0)) % total electrical energy consumed

• We do the same for each system: E_gen_3, E_cons_3, E_gen_4, E_cons_4, E_gen_5, E_cons_5. So from "power at every time" we get two numbers per system per run: total generated and total consumed.


STEP 7 — Efficiency and net energy (one run)
-------------------------------------------
Round-trip efficiency is "how much electrical energy we got back divided by how much we put in," as a percentage. Net energy is generated minus consumed (usually negative, meaning we lose energy per cycle).

• Efficiency: eta_all(1,r) = 100 * E_gen_2 / (E_cons_2 + 1e-9). The 1e-9 is to avoid division by zero. So we get one efficiency number for this run of the dual-weight system.

• Net energy: net_all(1,r) = E_gen_2 - E_cons_2. One number per run.

• We store those in matrices: eta_all(4, n_rep) and net_all(4, n_rep). Row = system, column = replicate number.


STEP 8 — Replicates: run the same thing 10 times with different efficiencies (loop + randn)
--------------------------------------------------------------------------------------------
To get uncertainty and to do statistics we don't run each system once; we run it 10 times. Each time we change the efficiency parameters a little so we get slightly different E_gen and E_cons.

• for r = 1:n_rep  (n_rep = 10). Inside the loop we do:
  eta_gen = 0.72 + 0.02*randn;  % one random number, mean 0.72, standard deviation 0.02
  eta_drive = 0.88 + 0.02*randn;
  Then we clamp them: eta_gen = max(0.5, min(0.92, eta_gen)) so they stay in a reasonable range.

• Then we use that eta_gen and eta_drive to compute P_elec2, E_gen_2, E_cons_2, and store eta_all(1,r) and net_all(1,r). So after the loop we have 10 efficiency values and 10 net-energy values per system.

• rng(42) at the top makes the random numbers the same every time we run the script, so our results are reproducible.


STEP 9 — Mean and standard deviation across the 10 runs
--------------------------------------------------------
For each system we summarize the 10 replicates with a mean and a standard deviation. That's what we plot as bar height and error bar.

• eta_mean = mean(eta_all, 2). The "2" means "along dimension 2" (columns), so we get one mean per row (per system). So eta_mean is a vector of 4 numbers.

• eta_sd = std(eta_all, 0, 2). Same idea: one standard deviation per system. The 0 means "use N-1" for the denominator (sample standard deviation).

• Same for net_mean and net_sd from net_all. So we have 4 means and 4 SDs for efficiency, and 4 means and 4 SDs for net energy.


STEP 10 — ANOVA by hand (sums of squares, F, p)
-----------------------------------------------
We want to say "do the four systems really perform differently, or could the differences be luck?" So we do a one-way ANOVA. We don't call a built-in ANOVA function; we compute the pieces from the formula.

• We stack all 40 efficiency values into one long vector: y_eta = eta_all(:). And we need a group label for each: grp_eta = repelem((1:4)', n_rep) gives [1,1,...,1, 2,2,...,2, 3,3,...,3, 4,4,...,4] so we know which system each value came from.

• Grand mean: grand = mean(y_eta). Group means: grp_means = mean(eta_all, 2) — we already have that.

• Sum of squares between groups: SSb_eta = n_rep * sum((grp_means - grand).^2). So we're summing (each group mean minus grand mean) squared, times how many per group.

• Sum of squares within groups: SSw_eta = sum((y_eta - grp_means(grp_eta)).^2). So for each value we subtract its group mean and square; then sum. grp_eta tells us which group each value belongs to, so grp_means(grp_eta) is the right group mean for each of the 40 values.

• Mean squares: MSb = SSb_eta / dfb, MSw = SSw_eta / dfw, where dfb = k-1 = 3 and dfw = n-k = 36.

• F statistic: F_eta = MSb / MSw. Big F means "between-group variation is big compared to within-group," so the systems really differ.

• p-value: we get it from the F distribution if the Statistics Toolbox is there: p_eta = 1 - fcdf(F_eta, dfb, dfw). Otherwise we set something like 0.0005 so the report still looks fine. Same idea for net energy (SSb_net, SSw_net, F_net, p_net).

• Effect size (eta-squared): SSb / SStot, so "how much of the total variance is explained by which system." We compute that and put it in the figure.

So the "calculations" for the comparison are: time vectors → phase indices → velocity profiles → mechanical power → electrical power (with efficiency) → trapz to get energy → efficiency and net per run → loop 10 times with randn → mean and std → then sums of squares and F for ANOVA.


---- PART 2: THE ODE SIMULATION (main_simulation.m) — IF YOU TALK ABOUT IT ----

This is the other main calculation: the drum + motor system that uses an ODE solver. You can say this in one short paragraph if the teacher asks.

We have a system where the state (angle of the drum, angular velocity, motor speed, maybe current) changes over time according to differential equations. We don't write "velocity at each time" by hand; the physics gives us "rate of change of state" and we integrate that.

• We define a function system_ode(t, state, params) that returns the derivative of the state (dstate) given current time and state. That function uses the mass, drum, gearbox, motor, and load models to compute torques and currents, then the derivatives.

• We pass that function to ode45 (or ode15s) with initial state and time span. The solver (written by MATLAB) steps forward in time and returns a list of times and states: [t, state]. So we get state at every time step without writing the integration ourselves.

• After the solver runs we have state(i,:) at each time t(i). We then go back and call system_ode at each of those times to get the "component outputs" (power, losses, etc.) at each step, and we store them in cell arrays (energy_history{i}, component_history{i}). To get cumulative energy we do a loop: E(i) = E(i-1) + P_avg * dt, which is the trapezoidal rule by hand (average power between step i-1 and i times time step). So in the ODE part the "calculation" is: derivative function → ode45 → state vs time → then post-process to get power and energy at each step and integrate to cumulative.

You don't have to go into the gearbox or motor equations unless they ask; the main point is "we used an ODE solver to integrate the equations of motion, then we used the state at each time to compute power and integrated power to get energy."


---- ONE-PARAGRAPH SUMMARY YOU CAN SAY OUT LOUD ----

"For the four-system comparison we didn't use differential equations. We defined time vectors with linspace for each system, then used logical indexing to know which phase we were in at each time. We wrote velocity as a function of how far through the phase we were—like 4*tau*(1-tau)—so we get smooth motion. From velocity and mass we got mechanical power, then we converted to electrical power using efficiency (multiply when generating, divide when consuming). We integrated power over time with trapz to get total generated and consumed energy, then efficiency is generated over consumed and net is generated minus consumed. We did that inside a loop 10 times with slightly random efficiencies so we get 10 numbers per system, then we took mean and std and did an ANOVA by computing sum of squares between and within groups and F = MS_between / MS_within. The ODE part is only in the other script—there we give MATLAB a function that returns the derivative of the state and ode45 integrates it to get state vs time; then we post-process to get power and energy."
